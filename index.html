フェーズ別行動整理

早川さんが提案準備から商談、既存顧客の深耕まで、実際にどのような行動をとっているかを6つのフェーズに整理しました。4つのクライアント案件の分析と、ご本人へのインタビュー内容に基づいています。

この資料のポイント

早川さんの提案準備の最大の特徴は、「施策」や「サービスメニュー」からではなく、顧客の事業構造から入ることです。IR資料を読み、市場を理解し、事業のKGIから逆算して「何を提案すべきか」の仮説を組み立てる。この「上から下へ一気通貫で下ろす」プロセスが、提案の説得力と受注率に繋がっています。

本資料では、この一連の行動を6つのフェーズに分解し、各フェーズで早川さんが実際に何をしているか、そしてご本人がどう考えているかを具体例とともにまとめています。

1

事業・顧客理解

顧客の事業構造を経営レベルで把握し、顧客の顧客・競合環境・自社の立ち位置を構造的に理解する。
「顧客のビジネスを顧客以上に理解する」ための基盤づくり。

早川さんの考え方（インタビューより）
基本的には広いところからだんだん下げていくっていうところです。全社の業績はどうで、全社的な視点で見ると経営課題って何で、というところを一旦見た上で、今回の対象事業ってどういう位置づけにあるのかを把握する。
お客さんの立ち位置って3C分析とかでどうなのか整理して、誰がお客様で、そのお客様ってカスタマージャーニー的にはどういう動き方するのかを整理して、そこに対してマーケティングのアプローチとして4P整理するとどんなことをやっていそうか。1から4まで踏まえると、どんな支援をするとうまくフィット感がありそうかっていうのを考える。
2026年4月15日 インタビューより
1-1 IR資料・財務データの調査
サンリオビーウィズ
顧客企業のIR資料（決算説明会資料、半期報告書、中期経営計画など）を読み込み、全社業績・事業セグメント・成長戦略を把握する。 サンリオ: Q3累計の決算説明会資料から全社業績とピューロランド単体売上を把握。
ビーウィズ: 半期報告書・中計数値・セグメント構成を把握。
1-2 市場構造の把握
全4案件
顧客が戦っている市場の規模・構造・トレンドを調査する。 サンリオ: IP経済圏と推し活市場（4.1兆円）の構造を把握。
パナソニック: ベトナム空調市場のPEST分析。10以上の外部ソースURLを明記。
東芝: ディスクリート半導体市場の3極構造。ビーウィズ: BPO市場（1兆1,000億円超）と生成AIの影響。
1-3 3C分析
全4案件
Customer（顧客の顧客）、Competitor（競合）、Company（顧客企業自身）の3つの視点で市場を整理する。全案件で実施しており、早川さんの「標準装備」。 サンリオ: 社内用の「深い版」と顧客提示用の「提案版」の2種類を作成。競合は「テーマパーク同士」ではなく「時間の競合 / 予算の競合 / 体験の競合」の3層で再定義。
1-4 ペルソナ設計
全4案件
顧客の顧客や意思決定者の具体的な人物像を設計する。案件によって詳しさを変えている。 東芝: 最も詳細。「深夜にデータシートを漁る半導体エンジニア」「EOLリスクを極端に嫌う購買担当者」と行動レベルまで描写。
パナソニック: ターゲット属性レベル。案件の目的に応じて深さを調整。
1-5 カスタマージャーニー設計
東芝
顧客の顧客が認知から購入に至るまでの行動ステップを設計する。 情報収集 → 技術探索 → 技術検証 → 試作 → 評価 → 見積 → 発注の7ステップ。各ステップに「リアル」と「デジタル」のタッチポイントを配置。
1-6 AIを使った段階的リサーチとセカンドオピニオン
サンリオ
AIに一度に全部聞くのではなく、段階的に深掘りする。さらに外部専門家からのフィードバックも取得し、分析の精度を高める。 Geminiで全社業績 → 事業部 → 市場定義 → 3C → 戦略仮説 → ペルソナ → KPIと15ターンで段階的に深掘り。AIの3C分析に対して「競合はテーマパークではない」「モードの違いでは」と自分の仮説を3回ぶつけて修正。さらに外部コンサルタントから3,000字のFBを取得し統合。
早川さんのAI活用の考え方
AIもゴリゴリ使うんですが、ポイントはAIが吐き出してきたことを、ちゃんと自分の言葉で喋れるように咀嚼するってところがすごく重要。
一気に答えを出させることもできるんですけど、1個1個一問一答形式でやっていった方がいい。アウトプットが出てきたものに対して「それってどういうこと？」みたいなのを壁打ちする中で、内容を全部咀嚼していける。
2026年4月15日 インタビューより
このフェーズの注目ポイント
  • 全4案件で「事業構造の上位理解」→「3C分析」を実施 ── 施策レベルから入っていない
  • 新規コンペ（サンリオ）ではIR + AI + 専門家FBまで動員。既存案件では深さを調整
  • AIを「答えを出す道具」ではなく「壁打ち相手」として使い、自分の仮説をぶつけて深化させている
  • 3C分析は全案件共通。ペルソナ・CJは案件特性に応じて深さを変えている
2

仮説構築

フェーズ1の分析を踏まえ、「何を提案すべきか」「何をKPIとすべきか」の仮説を組み立てる。
早川さんが「上から下に下ろしてくる」と表現する、提案準備の核心部分。

早川さんの考え方（インタビューより）
仮説の精度と言ったんですけど、もちろんそれは高ければ高いに越したことはないんだけども、多分より本質的には、仮に精度が高くなかった、外れちゃってたとしても、そこまで一生懸命自分たちのことを理解しようとしてくれたんだっていう態度とか姿勢みたいなものが多分伝わるはず。
AIが出してくる仮説って感覚的に結構ちょっと遠いことが多い感覚はあるので、ここはそうじゃなくてこっちじゃないみたいなところ、それこそ壁打ちした上で自分の中での腹落ち感・精度感を高めていく。
2026年4月15日 インタビューより
2-1 事業KGIから広告KPIまでの一気通貫導出
サンリオ
顧客の事業目標（KGI）を起点に、マーケティング施策のKPIまでを一本の線で繋ぐ。途中でプリンシプルのサービスメニューに引き寄せない。 「サンリオのKGI = 夢中時間の最大化 = LTVの最大化」
→ 「ピューロランド = 体験接点のひとつ」
→ 「広告のKPI = Sanrio+会員登録」
事業構造から広告KPIまでを、論理のチェーンで直線的に導出。
2-2 KPI候補の比較評価
サンリオ
KPIの候補を複数出し、評価軸を設けて構造的に比較する。 KPI候補4つ: Sanrio+会員登録 / 来場パスポート購入 / 予約時売上 / LTV
評価軸3つ: 事業KGIとの直結性 / 計測可能性 / 最大化余地
結論: 「Sanrio+会員登録」が最適。
2-3 事業KPIとWeb KPIの紐づけ
パナソニック
顧客の売上に至るファネル（見込み客が顧客になるまでのステップ）を全段数値化し、「プリンシプルが改善できるのはどこか」を特定する。 訪問者 → 問合せ → チャット → 送客 → 受注 → 単価 → 金額の各段を数値化。
「広告で直接動かせるのはここ」「CRMで改善できるのはここ」と、改善レバーをファネル上にマッピング。
2-4 顧客の課題を起点にした仮説構築
ビーウィズ
顧客から聞いた課題を分析し、その原因と解決策の仮説を立てる。 顧客の声「デジマ経由の受注単価・ROIが低い」
→ 仮説: ミドルファネル層（比較検討段階の見込み客）へのアプローチで、検討プロセスの早期から営業が介入 → 受注単価の向上。
2-5 事業計画からの定量目標逆算
ビーウィズ
顧客の事業計画の数値を出発点に、達成に必要なリード数やチャネル配分を逆算する。 財務構造（中計数値） → SQL目標 → 商談化率 → 受注率 → リード目標 → チャネル別目標と、事業目標から施策の数値目標までを逆算チェーンで設計。
2-6 仮説の数値検証（推定と事実の分離）
サンリオ
仮説に含まれる数値について、「これは推定」「これは要確認」を明確に区別する。 「セグメント利益率20%は要検証」「来場者数180万人も要確認」と、推定値にフラグを立てて事実確認を実施。
このフェーズの注目ポイント
  • 全ての仮説が「事業構造の上位」から下ろされている ── 施策アイデアから入っていない
  • KPI選定は候補比較型 ── 「なんとなく」ではなく評価軸を設けて選んでいる
  • 推定と事実を明確に分離し、推定のまま提案に載せない
  • 仮説の精度だけでなく「理解しようとする姿勢」が本質的に重要（早川さん談）
3

提案体制づくり

チームを編成し、提案準備を推進する。スケジュール設計、アジェンダ設計、作業分担、情報の流れの管理。

3-1 案件受領即日のスケジュール設計
サンリオ
案件を受けたその日のうちに、提案までのスケジュールを設計してチームに共有。 案件受領当日（4/23 11:55）に7ステップの仮スケジュールをSlackに投稿。先方からスケジュール変更があった際も当日中に対応。12:02（7分後）には翌日の社内MTGを自らセット。
3-2 MTGアジェンダの構造化
サンリオ
MTGの前に、ゴール・アジェンダ・時間配分・協議論点を事前に構造化して共有する。 45分構成で5項目に時間配分。協議論点を3つに絞り込んで事前提示:
a. ビジネス理解   b. 伴走支援の示し方   c. 短期施策の具体案
3-3 分担確定後の自担当即日完了
サンリオ
分担を決めた後、まず自分が当日中に担当パートを仕上げる。 MTG直後（15:37）に7パートの分担をテキストで確定。当日中にフォルダ作成・骨子格納・自担当パートを完了。
3-4 方針協議の段階的実施
ビーウィズ
方針の協議を1回で済ませず、複数回に分けて段階的に深化させる。 3/31と4/13の2回にわたり方針協議を実施。1回目で仮説提示 → 2回目で分析深化・戦略具体化。
このフェーズの注目ポイント
  • サンリオでは案件受領から2日以内に全体制が整っている
  • MTGは「ゴール・論点・時間配分」を事前構造化 ── 会議を走らせず設計している
  • 分担後に「まず自分がやって見せる」行動がある
4

提案資料づくり

フェーズ1〜2の分析・仮説を、顧客に提示する提案書に落とし込む。

4-1 提案資料の構造設計
サンリオ東芝
提案資料の全体構成を、案件の特性に合わせて設計する。 サンリオ: 与件整理 → 事業概況 → 提案概要（4段階STEP） → 見積 → スケジュール → 体制
東芝: 3C → ペルソナ → CJ → 施策（広告/SEO/解析） → 見積 → 体制
4-2 顧客課題の本質整理
サンリオ
複数の課題を少数の本質に集約する。個別の問題をバラバラに並べず、構造化して提示する。 複数の課題を「2つの判断のしづらさ」に集約:
1. 事業貢献の不透明さ（広告が事業にどう効いているかが見えない）
2. 運用のブラックボックス（何をやっているのか把握できない）
4-3 メディアプランニング・コンテンツ戦略の具体化
東芝ビーウィズ
「方向性」ではなく「実行計画」レベルまで具体的な施策案を提示する。 東芝: LP x KW x ターゲティングの11パターンマッピング表。3プラン（250万/400万/650万円）で数値試算。
ビーウィズ: 15本のコンテンツ企画案を認知/興味関心/比較検討の3段階で設計。
4-4 見積設計
サンリオ東芝ビーウィズ
「いくらか」だけでなく「どう使うか」まで設計する。 東芝: 6項目で通期1.1億円の見積。
ビーウィズ: 自身の工数（25h + 協議15h）まで含めた設計。「金額」と「投下リソース」の両面で提案。
早川さんの見積に対する考え方
初回提案の場合は複数、いずれにせよ見積もりも作ると思うんですけど、予算感はあんまり重要でないことの方が多い。予算1000万ですって言った時にプランとしては2000万、1000万、500万みたいなレンジで多分出す。
2026年4月15日 インタビューより
このフェーズの注目ポイント
  • 提案資料の起点は常に「事業構造の理解」と「仮説」── サービスメニューの紹介から始まっていない
  • ビーウィズのコンテンツ戦略は15本という具体性 ── 「方向性」ではなく「実行計画」
  • 見積は複数プランで提示し、予算ありきではなく価値ありきで設計
5

商談

直前の準備、商談の進行、商談後のアクション。

早川さんの商談に対する考え方
ポイントは要はヒアリング、聞く場ではなくて、こう考えてきたけどそれで合ってるかっていう確認する場みたいな立ち位置にする。
プレゼンの場自体は言ってみたらほぼ儀式みたいなもので、やらなきゃいけないからプロセスとしてやってるだけで、ここ自体はさして重要じゃなく、担当者とのすり合わせの時点で結論が見えているみたいなことにしておく。
アイスブレイクのネタとして、企業のできればその事業に関わるところ、何かアイスブレイクになるようなネタがあれば、そこから始めていく。リアルタイム性が重要で、相談開始前30分とか5分で見つけに行ける。
2026年4月15日 インタビューより
5-1 移動中の再考察と直前更新
サンリオ
準備が完了した後も、ギリギリまで最適化を続ける。 訪問に向かう移動中に「広告効果の可視化がキー」と判断し、該当スライドを直前更新。
5-2 アイスブレイクの仕込み
サンリオ
商談の冒頭で、個人的な接点を示すスライドを用意。 提案資料の冒頭に、家族でピューロランドを訪問した写真のスライドを追加。「この仕事に個人的にも関心がある」ことを自然に伝える。
5-3 商談を「仮説検証の場」として進行
サンリオ
一方的なプレゼンではなく、仮説を提示して先方の反応で検証する対話型の場にしている。 与件・リサーチ内容について先方との認識ギャップを確認。質疑応答を「仮説の検証」として活用。
5-4 商談後の即日行動
サンリオ
商談が終わったらその日のうちに次のアクションを起こす。 商談後に担当者と電話。スケジュール変更情報を即日社内共有。次回面談に向けた社内相談を依頼。
早川さんが考える「勝ちパターンの起点」
クリアすると行けたなというのは、最初の初手として、まずお客さんってどういう状況にあるんだっけっていうのをこちら側からしっかりある程度外から見えるところで把握して、そこで一定程度の精度の高い仮説を持ってくことによって、初回の時に少なくともその担当者自体はもうこちらの仲間にする。
2026年4月15日 インタビューより
このフェーズの注目ポイント
  • 「移動中に再考察 → 直前更新」── 準備完了後もギリギリまで最適化
  • 提案を「仮説」として位置づけ、商談を「仮説検証の場」にしている
  • プレゼンは「儀式」。勝負は事前の担当者とのすり合わせで決まる（早川さん談）
  • 商談後のアクション（電話・社内共有・次回準備）が全て即日
6

既存顧客の深耕

日常の伴走支援の中から、次の提案機会を生み出す行動。3社の既存案件から抽出。

6-1 「伴走支援」ポジショニングの継続的伝達
パナソニック
「プリンシプルは広告の運用代行ではなく、事業成長に向けた伴走支援をしている」というポジショニングを、日常のやり取りの中で折に触れて伝え続ける。
6-2 定期タイミングでの分析更新
パナソニック
年度替わりなどのタイミングで、PEST/3Cの読み合わせを定期的に実施する。提案時に一度やって終わりにしない。
6-3 KPI紐づけから次の仕事を生む
パナソニック
事業KPIとWeb KPIの紐づけを協議する中で、自然に次の仕事が生まれる。 KPI紐づけの方針協議 → その議論の流れでタグマネ・解析が「やるべきこと」として浮上 → 新たな案件として受注。
6-4 年間伴走提案としての包括分析
東芝
単発の施策提案ではなく、3C/ペルソナ/CJを含めた年間の伴走支援提案を実施。
6-5 事業計画からの逆算目標を「叩き」として提示
ビーウィズ
顧客の事業計画から逆算した目標値を「叩き台」として提示する。顧客が自社では作れない価値を提供する行動。 事業計画の予測数値からデジタルマーケティングの目標値を逆算試算し、「叩き」としてクライアントに提示して目線合わせ。
このフェーズの注目ポイント
  • パナソニックの「KPI協議 → 新規案件化」は深耕の具体的成功パターン
  • 既存顧客でも3C/PESTを「定期的に更新」── 一度やって終わりにしない
  • 「事業計画から逆算した数値」を叩きとして出すのは、顧客が自力では作れない付加価値

フェーズ間の繋がり

フェーズ1 → 2: 事業理解から仮説まで一気通貫

サンリオでは、事業構造の理解（フェーズ1）から広告KPIの仮説（フェーズ2）まで、「経営 → 事業 → マーケ → 施策」の一本道で繋がっている。途中でプリンシプルのサービスメニューに引き寄せる動きがない。

フェーズ1: 分析の深さを案件に合わせて調整

3C分析は全案件で行うが、ペルソナは「属性レベル」から「行動レベル」まで深さが異なる。CJ設計は東芝のみ。「全部やる」ではなく「必要な深さでやる」判断がある。

フェーズ3: 案件受領から2日以内に体制構築

サンリオでは、スケジュール設計 → MTG設定 → チーム指示 → アジェンダ設計 → 分担確定 → 自担当完了が2日以内に完了している。

フェーズ6 → 2: 日常の伴走から仮説構築を経て次の仕事へ

パナソニックの「定期PEST/3C読み合わせ → KPI紐づけ → 新規案件化」は、フェーズ6（日常の深耕）がフェーズ2（仮説構築）を経由して新たな仕事に繋がるパターン。
