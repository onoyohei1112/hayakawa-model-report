2事業課題とKPIをつなぐ、一本の線

早川さんの仕事の核心は、顧客が本当に追うべき事業課題（メインイシュー）と、今回動かすべき指標（KPI）を特定し、その2つを論理でつなぐことにある。両者が一本の線で結ばれているのが理想の状態だが、それ以前に、「事業課題は何か」「追うべきKPIは何か」を自ら見極める力こそが、最も難しく、最も差がつくところだ。新規の案件でも、既存顧客への提案でも、この構造は変わらない。

「仮説」の定義（R8アカデミー、2026年7月）

ここで言う仮説とは、「プリンシプルから何を提案したら、顧客の課題を解決できるのか」を、1つの文章、長くても2〜3分で語れる形にまとめたものだ。優先度の高い事業課題は何か、それに対してどんな打ち手が考えられるか、という2つの要素をセットで含む。課題が分かるだけでは顧客は動けないため、解決策と必ずセットにする。初回商談は、この仮の説が合っているかを確認する場として使う。

早川さん本人の言葉

顧客のRFP（提案依頼書）に書かれているのは、本当のメインイシューにぶら下がる「サブイシューのごく一部」にすぎない。顧客自身もメインイシューとの接続に気づいていないことが多い。そこに気づかせ、顧客が社内で上席に「雄弁に説明できる」状態（＝チャンピオン化）にすることが、最も良い状態。

3判断を支える6つの原則

早川さんの判断を貫いている原則を6つに整理した。同じリサーチや資料作成であっても、この6つが働いているかどうかで、提案の質は大きく変わる。この6つは、2026年7月のR8アカデミーで早川さん本人が全社向けに直接解説した内容と一致している。

1

顧客が言った言葉を起点に組む

自社サービスの説明から入らない。顧客がIR・プレス・RFPで繰り返し使う「言葉」を拾い、そこからすべての提案を組み立てる。逆に、自社側の用語（CPA・CPC・ROAS・GTMなど）は、顧客が日常的に使っていない限り極力使わない。使った瞬間に相手との間に壁ができ、その解消は難しいため。

実例：サンリオの「夢中時間」をピン留めし、そこからKPIまで論理を通した。
2

メインイシューに「気づかせる」

RFPは氷山の一角。その奥にある本当の事業課題を特定し、顧客自身が気づいていない接続を見せることで、顧客が社内で説明できる状態をつくる。

実例：東芝で「東芝さん以上に東芝さんの事業背景を考える」ことで、「自分たちより考えている」と刺さった。
3

「自分が先方担当者なら」で絞る

競合の切り方も、課題の絞り込みも、判断基準は一貫してこれ。「自分がその担当者だったら、どれに一番納得し、改善の余地を感じるか」。

実例：課題を「2つの判断のしづらさ」に集約するなど、複数の切り口から"自分が担当者なら一番納得するもの"を選んだ。
4

抽象度を上げて、提案を大きくする

目先のサブイシューに留まると提案は小さく終わる。上位の課題（メインイシュー）に接続するほど、打ち手の範囲と提案金額が大きくなる。

実例：「CPAを改善したい」「SEO流入が減ったので対策したい」のような具体の依頼にも、聞かれたことに答えたうえで「それは何のためか」を1段上げて考える。より本質的な課題と別の解決策が見え、提案の幅が広がる。これがクロスセルの起点にもなる。
5

オタク的な関心と、意図的なキャラ作り

顧客とその事業に本気で興味を持つ。さらにその関心を商談の場で意図的に演出し、信頼を獲得する。

実例：「子供とサンリオ」のエピソード（事実）を膨らませて、興味のあるキャラクターを演じた。
6

AIで広げ、自分で噛み砕く

AIで大量に情報・仮説を出した後、自分が人に語れる粒度まで噛み砕いて落とし込む。出力しただけでは使えない、という前提に立つ。

4勝ち案件と失注案件の違い

この「事業課題とKPIの接続」が、実際に勝敗を分けているのか。早川さん自身の受注した案件と失注した案件を対比すると、その差がはっきりと表れる。

失注事例

KEEN（サンダルメーカー）の案件

当時は、いまのようにメインイシューを作り込み、現場のKPIに接続する、ということを深くやっていなかった。会社のケイパビリティやサービスの特徴を説明しただけの薄い提案に終わった。

顧客の質問には答えられたが、こちらから積極的にメインイシューへ接続する話をしなかった。振り返ると「分かれ目」のような瞬間はなく、全体が"ふわっと"始まり、"ふわっと"終わった。

この対比からわかること

受注した案件にあり、失注した案件になかったのは、メインイシューの深掘りと、KPIへの接続そのものだった。ここまで述べてきた考え方の有無が、そのまま勝敗として表れている。

早川さんの戦い方

  • 広告運用の"巧拙"だけでは戦わない。「100を150にする」運用改善型の勝負より、「なぜこの100が必要か（意義）」を事業課題に接続して示す案件で強い。
  • 「ずらし」で対応する。広告運用で150を出せなくても、他施策と組み合わせて合計300以上の価値を提案する。土俵を運用単体からずらす。
  • 構造的な難所：広告担当者レベルから来た案件は、上位の意思決定者に話を上げにくい。意思決定者へ接近する「エンタープライズ・セールス」を、早川さんは意図せず自然にやっていた可能性がある（＝再現対象として要注目）。
早川さん本人の言葉（R8アカデミー、2026年7月）

「昔はここまで見られておらず、目の前に来た与件にのみ直接答えて提案していた。すると、そのままふわっと流れて、返事が来ない・失注になるということがあった。勝てている案件と失注した案件の違いは、このプロセスをちゃんとやれているかどうかが一番大きい。」

5既存顧客の深耕と、与件の出し切り

早川さんは、既存顧客の"深耕"を「後から新しい与件を生む」こととは捉えていない。初期提案の段階で、その時点に存在する与件をすべて洗い出し、"複合提案"として出し切るのが基本だ。だから、後から与件や次のSQLが出てくるのは、①新規提案の時点で課題を特定しきれなかったか、②状況が変化して新たに発生したかのどちらか、と捉える方が有効だ。ここでも効くのは、前半で述べた「メインイシューとKPIを特定しきる力」だ。

初期提案の段階で、その時点の与件を出し切る

顧客の事業を深く理解し、その時点に存在する課題を洗い出して、必要な打ち手を束（複合）で提案する。単発の施策売りにせず、事業課題に対する一式で出し切るからこそ、取りこぼしが減る。

先回りして「叩き台」を持ち込む

ビーウィズでは、共有されにくいはずの事業計画の数値を、決算書から自分で作って提案前のすり合わせに自主的に持ち込んだ。情報をもらうのを待たず、課題を可視化する材料を自分でつくって持っていく。

状況変化を捉え直す「ブリッジの場」（②を拾う仕組み）

折に触れて「事業KGIと通常業務をブリッジする協議の場」を設ける。ここで、状況変化で新たに生じた課題（②）を早期に捉え直す。やるべきことが自社のケイパビリティと一致すれば、自然にクロスセルの与件になる。

この「場」には型がある。年間プレゼンテーション（ネプレ）、クォーターレビュー（四半期の振り返り）、そして経営層の来訪に合わせた挨拶の場（「社長が来ているのでご挨拶に」と設定した場のアジェンダに、事業課題と普段の業務をつなぐ話を組み込む）などだ。定例の延長を待つのではなく、こうした場を意図的に作りにいくこと自体が、R8アカデミーで早川さんが勧めた具体的な打ち手だ。

東芝の例

東芝は間接部門であるがゆえに、デジタルマーケティングの成果が見えにくかった。だからこそ「何のためにこれをやっているのか」の意味を整備し続けることが効く。これを怠ると、担当部長の交代など状況変化で「意味が乗っている瞬間」に取引が終わる。早川さんは定例会にこの「事業背景の見合わせ」を織り込み、状況変化を早期に課題として特定して次につなげている。

①の分析を踏まえ、「何を提案すべきか」「何をKPIとすべきか」の仮説を組み立てる。早川さんが「上から下に下ろしてくる」と表現する、提案準備の核心部分。

仮説の精度は高いに越したことはないが、より本質的には、仮に外れていたとしても、そこまで一生懸命自分たちのことを理解しようとしてくれたという態度や姿勢が伝わるはず。AIが出してくる仮説は感覚的に少し遠いことが多いので、壁打ちしたうえで自分の中での腹落ち感・精度感を高めていく。

2-1事業KGIから広告KPIまでの一気通貫導出サンリオ

顧客の事業目標（KGI）を起点に、マーケティング施策のKPIまでを一本の線でつなぐ。途中で自社のサービスメニューに引き寄せない。

「サンリオのKGI＝夢中時間の最大化＝LTV最大化」→「ピューロランド＝体験接点のひとつ」→「広告KPI＝Sanrio+会員登録」。事業構造から広告KPIまでを論理のチェーンで直線的に導出。

2-2KPI候補の比較評価サンリオ

KPIの候補を複数出し、評価軸を設けて構造的に比較する。

KPI候補4つ（Sanrio+会員登録／来場パスポート購入／予約時売上／LTV）× 評価軸3つ（事業KGIとの直結性／計測可能性／最大化余地）で比較し、「Sanrio+会員登録」を選定。

2-3事業KPIとWeb KPIの紐づけパナソニック

顧客の売上に至るファネルを全段数値化し、「プリンシプルが改善できるのはどこか」を特定する。

訪問者 → 問合せ → チャット → 送客 → 受注 → 単価 → 金額の各段を数値化。「広告で直接動かせるのはここ」「CRMで改善できるのはここ」と、改善レバーをファネル上にマッピング。

2-4顧客の課題を起点にした仮説構築ビーウィズ

顧客から聞いた課題を分析し、その原因と解決策の仮説を立てる。

顧客の声「デジマ経由の受注単価・ROIが低い」→ 仮説「ミドルファネル層（比較検討段階の見込み客）へのアプローチで、検討プロセスの早期から営業が介入すれば、受注単価が向上する」。

2-5事業計画からの定量目標逆算ビーウィズ

顧客の事業計画の数値を出発点に、達成に必要なリード数やチャネル配分を逆算する。

財務構造（中計数値） → SQL目標 → 商談化率 → 受注率 → リード目標 → チャネル別目標と、事業目標から施策の数値目標までを逆算チェーンで設計。

2-6推定値と事実の区別サンリオ

仮説に含まれる数値について、「これは推定」「これは要確認」を区別して扱う。

「セグメント利益率20%は要検証」「来場者数180万人も要確認」と推定値にフラグを立てて事実確認を実施。なお6月のインタビューでは、フェルミ推定の数値を提案に載せること自体はあるとの補足があり、載せてよい数値の厳密な線引きルールがあるわけではない。
このフェーズの注目ポイント
  • すべての仮説が事業構造の上位から下ろされている。施策アイデアから入っていない
  • KPI選定は候補比較型。「なんとなく」ではなく評価軸を設けて選んでいる
  • 推定値にはフラグを立てて扱う（提案に載せない、という厳密なルールではない）
  • 仮説の精度だけでなく「理解しようとする姿勢」が本質的に重要（本人談）

③提案体制

チームを編成し、提案準備を推進する。スケジュール設計、アジェンダ設計、作業分担、情報の流れの管理。

3-1案件受領即日のスケジュール設計サンリオ

案件を受けたその日のうちに、提案までのスケジュールを設計してチームに共有する。

案件受領当日の11:55に7ステップの仮スケジュールをSlackへ投稿。12:02（7分後）には翌日の社内MTGを自らセット。先方からのスケジュール変更にも当日中に対応。

3-2MTGアジェンダの構造化サンリオ

MTGの前に、ゴール・アジェンダ・時間配分・協議論点を構造化して共有する。

45分を5項目に時間配分。協議論点を3つ（a. ビジネス理解　b. 伴走支援の示し方　c. 短期施策の具体案）に絞り込んで事前提示。

3-3分担確定後の自担当即日完了サンリオ

分担を決めた後、まず自分が当日中に担当パートを仕上げる。

MTG直後の15:37に7パートの分担をテキストで確定。当日中にフォルダ作成・骨子格納・自担当パートを完了。

3-4方針協議の段階的実施ビーウィズ

方針の協議を1回で済ませず、複数回に分けて段階的に深める。

3/31と4/13の2回にわたり方針協議を実施。1回目で仮説を提示し、2回目で分析の深化と戦略の具体化を行う。
このフェーズの注目ポイント
  • サンリオでは案件受領から2日以内に全体制が整っている
  • MTGは「ゴール・論点・時間配分」を事前に構造化しており、会議を走らせるのではなく設計している
  • 分担後に「まず自分がやって見せる」行動がある

④提案資料

①②の分析・仮説を、顧客に提示する提案書に落とし込む。

4-1提案資料の構造設計サンリオ東芝

提案資料の全体構成を、案件の特性に合わせて設計する。

サンリオ: 与件整理 → 事業概況 → 提案概要（4段階STEP） → 見積 → スケジュール → 体制。東芝: 3C → ペルソナ → カスタマージャーニー → 施策（広告／SEO／解析） → 見積 → 体制。

4-2顧客課題の本質整理サンリオ

複数の課題を少数の本質に集約する。個別の問題をバラバラに並べず、構造化して提示する。

複数の課題を「2つの判断のしづらさ」に集約。1. 事業貢献の不透明さ（広告が事業にどう効いているかが見えない）　2. 運用のブラックボックス（何をやっているのか把握できない）。

4-3施策の実行計画レベルまでの具体化東芝ビーウィズ

「方向性」ではなく「実行計画」レベルまで具体的な施策案を提示する。

東芝: LP×キーワード×ターゲティングの11パターンのマッピング表。3プラン（250万／400万／650万円）で数値試算。ビーウィズ: 15本のコンテンツ企画案を認知／興味関心／比較検討の3段階で設計。

4-4見積設計サンリオ東芝ビーウィズ

「いくらか」だけでなく「どう使うか」まで設計する。

東芝: 6項目で通期1.1億円の見積。ビーウィズ: 自身の工数（25時間＋協議15時間）まで含めて設計し、金額と投下リソースの両面で提案。
早川さん本人の言葉（2026年4月インタビュー、見積について）

初回提案の場合は、いずれにせよ見積もりを複数作る。予算感はあまり重要でないことのほうが多い。予算1,000万円と言われたら、プランとしては2,000万・1,000万・500万のようなレンジで出す。

このフェーズの注目ポイント
  • 提案資料の起点は常に「事業構造の理解」と「仮説」。サービスメニューの紹介から始まっていない
  • ビーウィズのコンテンツ戦略は15本という具体性。「方向性」ではなく「実行計画」を出している
  • 見積は複数プランで提示し、予算ありきではなく価値ありきで設計している

⑤商談

直前の準備、商談の進行、商談後のアクション。

早川さん本人の言葉（2026年4月インタビュー）

ポイントは、ヒアリング（聞く場）ではなく、「こう考えてきたが、それで合っているか」を確認する場という立ち位置にすること。プレゼンの場自体はほぼ儀式のようなもので、担当者とのすり合わせの時点で結論が見えている状態にしておく。アイスブレイクのネタは、できれば先方の事業に関わるもの。リアルタイム性が重要で、商談開始前の30分や5分で見つけに行ける。

5-1移動中の再考察と直前更新サンリオ

準備が完了した後も、直前まで最適化を続ける。

訪問に向かう移動中に「広告効果の可視化がキー」と判断し、該当スライドを直前に更新。

5-2アイスブレイクの仕込みと関心の演出サンリオ

商談の冒頭で、個人的な接点を示す仕込みを用意する。先方の事業への興味・関心は、事実を土台に意図的に膨らませて伝えている。

提案資料の冒頭に、家族でピューロランドを訪問した写真のスライドを追加。「この仕事に個人的にも関心がある」ことを自然に伝える。6月のインタビューでは、事実として存在するエピソードをしっかり膨らませて伝える「キャラ作り」を意図的に行っているとの補足があった。

5-3商談を「仮説検証の場」として進行サンリオ

一方的なプレゼンではなく、仮説を提示して先方の反応で検証する対話型の場にする。

与件・リサーチ内容について先方との認識ギャップを確認。質疑応答を「仮説の検証」として活用。

5-4商談後の即日行動サンリオ

商談が終わったらその日のうちに次のアクションを起こす。

商談後に担当者と電話。スケジュール変更情報を即日社内共有。次回面談に向けた社内相談を依頼。
早川さん本人の言葉（2026年4月インタビュー、勝ちパターンの起点）

まずお客さんがどういう状況にあるのかを、外から見えるところでしっかり把握し、一定程度精度の高い仮説を持っていくことによって、初回の時点で、少なくとも担当者をこちらの仲間にする。

このフェーズの注目ポイント
  • 「移動中に再考察 → 直前更新」と、準備完了後も直前まで最適化を続けている
  • 提案を「仮説」として位置づけ、商談を「仮説検証の場」にしている
  • プレゼンは「儀式」であり、勝負は事前の担当者とのすり合わせで決まる（本人談）
  • 商談後のアクション（電話・社内共有・次回準備）がすべて即日

⑥既存顧客の深耕

日常の伴走支援の中から、次の提案機会を生み出す行動。3社の既存案件から抽出。

6-1「伴走支援」ポジショニングの継続的伝達パナソニック

「プリンシプルは広告の運用代行ではなく、事業成長に向けた伴走支援をしている」というポジショニングを、日常のやり取りの中で折に触れて伝え続ける。

6-2定期タイミングでの分析更新パナソニック

年度替わりなどのタイミングで、PEST・3Cの読み合わせを定期的に実施する。提案時に一度やって終わりにしない。

6-3KPI紐づけから次の仕事を生むパナソニック

事業KPIとWeb KPIの紐づけを協議する中で、自然に次の仕事が生まれる。

KPI紐づけの方針協議 → その議論の流れでタグマネジメント・解析が「やるべきこと」として浮上 → 新たな案件として受注。

6-4年間伴走提案としての包括分析東芝

単発の施策提案ではなく、3C・ペルソナ・カスタマージャーニーを含めた年間の伴走支援提案を実施する。

6-5事業計画からの逆算目標を「叩き台」として提示ビーウィズ

顧客の事業計画から逆算した目標値を「叩き台」として提示する。顧客が自社では作れない価値を提供する行動。

事業計画の予測数値からデジタルマーケティングの目標値を逆算試算し、叩き台としてクライアントに提示して目線合わせを行う。
このフェーズの注目ポイント
  • パナソニックの「KPI協議 → 新規案件化」は、深耕の具体的な成功パターン
  • 既存顧客でも3C・PESTを定期的に更新しており、一度やって終わりにしない
  • 「事業計画から逆算した数値」を叩き台として出すことは、顧客が自力では作れない付加価値になっている

フェーズ間のつながり

① → ②　事業理解から仮説まで一気通貫

サンリオでは、事業構造の理解（①）から広告KPIの仮説（②）まで、全社 → 事業 → マーケティング → 施策の一本道でつながっている。途中でプリンシプルのサービスメニューに引き寄せる動きがない。

①　分析の深さを案件に合わせて調整

3C分析は全案件で行うが、ペルソナは「属性レベル」から「行動レベル」まで深さが異なり、カスタマージャーニー設計は東芝のみ。「全部やる」ではなく「必要な深さでやる」という判断がある。

③　案件受領から2日以内に体制構築

サンリオでは、スケジュール設計 → MTG設定 → チーム指示 → アジェンダ設計 → 分担確定 → 自担当完了が、2日以内に完了している。

⑥ → ②　日常の伴走から仮説構築を経て次の仕事へ

パナソニックの「定期的なPEST・3C読み合わせ → KPI紐づけ → 新規案件化」は、日常の深耕（⑥）が仮説構築（②）を経由して新たな仕事につながるパターン。この考え方は概念編の「既存顧客の深耕と、与件の出し切り」の章で詳しく扱う。

目標ラインと時間の目安（2026年7月 R8アカデミー）

目標ライン。①事業理解のゴールは、「顧客の担当者に代わって、顧客の事業を社内のメンバーにプレゼンできるレベル」だ。「顧客よりも顧客に詳しくなる」の手前にある、乗り越えやすい現実的なハードルとして、まずここを目指す。顧客のことをよく知らない相手（同僚・上司、なんなら家族）に、自分の言葉で噛み砕いて説明できる状態が目安だ。

時間の目安。①は30分〜長くても1時間で一定の水準までできる。前半30分で❶〜❺をざっと調べ、後半30分で調べた内容を自分の言葉に噛み砕く配分だ。AIには一括で出させず、1問1答で各プロセス2〜3往復の壁打ちをする。慣れないうちは1〜2時間かかるが、慣れると30分程度まで縮む。まずは10分でライトに始めてよい。毎朝、ニュースで見かけた企業のIRを1社見る、という日々のトレーニングも有効だ。

情報が少ない企業（未上場など）の場合。AIへの質問とフェルミ推定で、事業規模の概算はつかめる。それでも足りなければ、同じビジネスモデルの企業群に共通する課題を軸に、仮説を組み立てる（R8アカデミー質疑より）。

    メインイシューとKPI、そして⑤の接続の一文を、ノートかホワイトボードに書き出す。それを声に出して人に説明してみる。聞き手は、できれば経営経験のある人・自分の事業を持つ人（最低でもマネジメント経験者）。「このメインイシューとKPIを、こう繋げました」と話し、相手が腹落ちするか。詰まる／相手がピンと来なければ、まだ接続が弱い。
